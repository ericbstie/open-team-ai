# Skills

Engineering skills vendored from [mattpocock/skills](https://github.com/mattpocock/skills) (MIT licensed), flattened one level so each is directly discoverable under `.claude/skills/<name>/SKILL.md`.

- **ask-matt** — Ask which skill or flow fits your situation.
- **code-review** — Two-axis review of the diff since a fixed point: Standards and Spec.
- **codebase-design** — Shared discipline and vocabulary for designing deep modules.
- **diagnosing-bugs** — Disciplined diagnosis loop for hard bugs and performance regressions.
- **domain-modeling** — Actively build and sharpen a project's domain model.
- **grill-with-docs** — Grilling session that also builds your project's domain model.
- **implement** — Build the work described by a spec or set of tickets.
- **improve-codebase-architecture** — Scan a codebase for deepening opportunities.
- **prototype** — Build a throwaway prototype to answer a design question.
- **research** — Investigate a question against high-trust primary sources.
- **resolving-merge-conflicts** — Work through an in-progress git merge or rebase conflict.
- **setup-matt-pocock-skills** — Configure this repo for the engineering skills.
- **tdd** — Test-driven development with a red-green-refactor loop.
- **to-spec** — Turn the current conversation into a spec and publish it to the issue tracker.
- **to-tickets** — Break any plan, spec, or conversation into a set of tracer-bullet tickets.
- **triage** — Move issues through a state machine of triage roles.
- **wayfinder** — Plan a huge chunk of work as a shared map of decision tickets.
